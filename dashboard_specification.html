📋 1. ダッシュボード概要 (Dashboard Overview)

🎯 目的 (Purpose)

OS部門の業務パフォーマンスと財務状況をリアルタイムで可視化し、ライセンスコストを考慮した実収益を管理するための総合ダッシュボード。

主要機能

  • ✅ 計画時間と実績時間のトラッキング
  • ✅ CADライセンスコストの管理と計算
  • ✅ 総収益（Gross）と純収益（Net）の可視化
  • ✅ プロジェクト別パフォーマンス分析
  • ✅ 損益分岐点とコスト分析
  • ✅ 日本通貨フォーマット（¥ and 万円）対応

ユーザー

  • Primary: OS部門マネージャー（編集・管理権限）
  • Secondary: 経営層（閲覧専用）
  • Technical Level: 非技術者、Excel経験者

🔄 8. データフロー

Data Sources

Database Tables:
├── projects (プロジェクト基本情報)
│   ├── id, company, projectType, software
│   └── created_at, updated_at
│
├── project_hours (工数データ)
│   ├── project_id, period, month, type
│   ├── hours (integer)
│   └── created_at, updated_at
│
├── license_costs (ライセンスコスト)
│   ├── period, number_of_computers
│   ├── price_per_computer, total_cost
│   └── notes
│
└── settings (システム設定)
    ├── unit_price (時間単価)
    └── active_periods (表示期間)

Calculation Flow

Step-by-Step Calculations:
  1. Get Base Data
    • totalPlanHours = sum(project_hours where type='plan')
    • totalActualHours = sum(project_hours where type='actual')
    • unitPrice = settings.unit_price
    • totalLicenseCost = license_costs.total_cost
  2. Calculate Basic Metrics
    • achievementRate = (totalActualHours / totalPlanHours) × 100
  3. Calculate Gross Revenue
    • grossRevenuePlan = totalPlanHours × unitPrice
    • grossRevenueActual = totalActualHours × unitPrice
  4. Calculate Net Revenue
    • netRevenuePlan = grossRevenuePlan - totalLicenseCost
    • netRevenueActual = grossRevenueActual - totalLicenseCost
  5. Calculate Profit Margins
    • profitMarginPlan = (netRevenuePlan / grossRevenuePlan) × 100
    • profitMarginActual = (netRevenueActual / grossRevenueActual) × 100
  6. Calculate Cost Metrics
    • licenseCostPerHour = totalLicenseCost / totalPlanHours
    • netHourlyRate = unitPrice - licenseCostPerHour
    • breakEvenHours = totalLicenseCost / unitPrice

💻 10. 実装メモ

Critical Requirements

⚠ Must-Have Features:
  • Real-time calculation of all metrics when license cost changes
  • 万円 conversion: amount / 10000 with 1 decimal place
  • Negative value styling: Red color for net revenue < 0
  • All cards in horizontal layout (4 per row on desktop)
  • Cost analysis metrics in horizontal layout (3 metrics side-by-side)
  • Responsive grid that adapts to screen size

State Management

React State Structure: 
const [licenseConfig, setLicenseConfig] = useState({
  numberOfComputers: 7,
  pricePerComputer: 2517143,
  // totalCost calculated dynamically
});

const [projects, setProjects] = useState([...]); // From Supabase
const [unitPrice, setUnitPrice] = useState(2300); // From settings
const [activePeriod, setActivePeriod] = useState('H2-2024');

// Computed values
const totalLicenseCost = useMemo(() => 
  licenseConfig.numberOfComputers * licenseConfig.pricePerComputer,
  [licenseConfig]
);

const calculations = useMemo(() => ({
  totalPlanHours: ...,
  totalActualHours: ...,
  grossRevenuePlan: ...,
  grossRevenueActual: ...,
  netRevenuePlan: ...,
  netRevenueActual: ...,
  // ... all other metrics
}), [projects, unitPrice, totalLicenseCost]);

Performance Optimization

  • Use useMemo for all calculations
  • Debounce license cost inputs (500ms)
  • Lazy load project table if > 50 projects
  • Virtual scrolling for large tables

Accessibility

  • ARIA labels for all input fields
  • Keyboard navigation support
  • Screen reader announcements for value changes
  • High contrast mode support
  • Focus indicators on interactive elements
